# vault-knowledge

用户的个人知识库工具。库是可复用知识和经验的唯一来源。

> **安装说明**：使用前请将本文件中所有 `<VAULT>` 替换为你的实际 vault 路径，例如 `/Users/yourname/Documents/Obsidian Vault`

## 🔥 沉淀准则（最高优先级，覆盖所有其他规则）

**vault 是个人经验库，不是百科。**

1. **每条笔记必须有"我"**——我的认知层级 / 我的思考 / 我的应用 / 我抽象的方法论。没有"我"的不进 vault
2. **不编**——用户没问过 / 没学过 / 没经历过的内容**不要主动补全**
3. **百科只留用户真问过的**——"X 是什么"类内容，只保留用户真问过且理解了的部分，不为完整性扩写
4. **经验 = 知识 + 我的理解**——两者都要，但都基于真实问答
5. **抗拒"完整知识地图"**——允许碎片、允许只懂一半、允许长期空着

### 反模式（绝对避免）

- ❌ 抄百科 / GitHub README / 官方文档进 vault
- ❌ 看到主题就扩写"完整列表 / 完整对比 / 完整流程"
- ❌ 替用户预先补"我的视角"段落
- ❌ 写半年就过时的数据（模型清单、价格表、版本号）

### 4 类内容 → 模板 → 目录

| 内容 | 模板 | 目录 |
|---|---|---|
| 学习经验 | `Experience.md` | `10-Knowledge/经验/<领域>/` |
| 应用实践 | `Iteration.md` | `20-Products/<产品>/20-Iterations/` |
| 抽象方法论 | `Methodology.md` | `10-Knowledge/方法论/` |
| 概念背景（最少量）| `Knowledge.md` | `10-Knowledge/经验/<领域>/`（只写"我理解到的层级"）|

## 通用规则

1. **永远先搜后写**——写任何文件前，先用 Glob/Grep 在库里检查是否已有相关条目
2. **复用优先**——已有条目就更新它，别建第二个同主题文件
3. **遵守库的 `CLAUDE.md`**——第一次进库时读 `<VAULT>/CLAUDE.md`，里面有目录约定和命名规范
4. **用模板**——新建知识/迭代/规划时，去 `99-Templates/` 拿对应模板再填
5. **双向链接**——条目里引用其他主题用 `[[名称]]`，目标暂不存在也写
6. **Frontmatter 必填**：`type`、`created`、`updated`、`tags`，日期格式 `YYYY-MM-DD`
7. **写完报告**：告诉用户写到了哪个文件、用的哪个模板、关联了哪些已有条目

## 流程：沉淀

触发：用户明确说"沉淀"/"归档"。

```
1. 一句话向用户确认要沉淀的主题（避免误归档闲聊）
2. Glob 搜 "10-Knowledge/**/*.md" 和 "20-Products/**/*.md"，看有没有相关条目
3a. 找到了 → 提议追加到该条目（"我准备把这部分加到 [[已有条目]] 里，OK 吗？"）
3b. 没找到 → 决定路径：
    - 学习经验（"我学 X 学到哪一档"）→ 10-Knowledge/经验/<领域>/<主题>.md（用 Experience.md）
    - 抽象方法论 → 10-Knowledge/方法论/<主题>.md（用 Methodology.md）
    - 纯概念笔记（少量）→ 10-Knowledge/经验/<领域>/<主题>.md（用 Knowledge.md，只写"我理解到的层级"）
    - 迭代方案 → 20-Products/<产品>/20-Iterations/YYYY-MM-DD-<英文短名>.md（用 Iteration.md）
    - 产品规划 → 20-Products/<产品>/00-Roadmap.md（用 Roadmap.md）
    - 拿不准归类 → 00-Inbox/<主题>.md，注明"待归档"
4. Read 模板 → 复制结构 → 填入对话提炼的内容
5. 末尾"来源"段记下当前对话日期
6. Write 文件
7. 告诉用户路径，并问要不要补 [[]] 链接
```

## 流程：先搜后答

触发：用户提问，主题可能在库的覆盖范围。

```
1. 用 Grep 在库里全文搜关键词（用户问题里的核心名词）
2. 命中 ≥ 1 个相关文件：
   - Read 命中文件，把答案锚定在已有笔记上
   - 引用："你之前在 [[X]] 里记过…"
   - 答完问："要不要根据这次对话更新 [[X]]？"
3. 没命中：
   - 正常回答
   - 答完主动问："这块要沉淀吗？"（不强推）
```

## 流程：综合

触发："整理 X" / "总结我的 Y 产品"。

```
1. Glob 圈定范围（如整理项目 → 20-Products/**/00-Roadmap.md + **/Interview*.md）
2. Read 所有相关文件
3. 按需组织答案（项目列表、对比表等）
4. 不直接落盘，先给用户看综合稿
5. 用户认可后再决定要不要写到 vault
```

## 反模式

- ❌ 没搜就建新文件（会产生重复）
- ❌ 把闲聊也沉淀（"沉淀"必须用户明示）
- ❌ 文件名用空格、特殊符号
- ❌ 改用户已有内容前不告诉他
